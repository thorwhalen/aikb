aikb.sync

Bidirectional sync engine for MutableMapping stores.

Provides push/pull/clone/sync/status operations that work with any collections.abc.MutableMapping — plain dicts, aikb KnowledgeFiles, dol stores, or anything else that implements the mapping protocol.

The sync engine is store-agnostic: it never imports aikb.base or any provider. All it needs is __getitem__, __setitem__, __delitem__, and __iter__.

Zero external dependencies — stdlib only (hashlib, json, dataclasses, enum).

aikb.sync.reconcile(snap_a: dict[str, str], snap_b: dict[str, str], *, manifest: dict[str, str] | None = None) → list[SyncAction][source]

Three-way reconciliation between snapshots A, B and a manifest.

If manifest is None, falls back to two-way comparison (cannot distinguish “created on A” from “deleted on B”).

Returns a list of SyncAction instances — one per filename across the union of all three sets.

aikb.sync.sync(a: ~collections.abc.MutableMapping, b: ~collections.abc.MutableMapping, *, manifest: dict[str, str] | None = None, manifest_path: str | ~pathlib.Path | None = None, on_conflict: ~aikb.sync.ConflictPolicy | ~typing.Callable[[~aikb.sync.SyncAction], ~aikb.sync.SyncAction] = ConflictPolicy.RAISE, dry_run: bool = False, hash_fn: ~typing.Callable[[str], str] = <function content_hash>) → list[SyncAction][source]

Bidirectional sync with three-way reconciliation.

Requires a manifest (or manifest_path) for correct conflict detection. If neither is provided, uses an empty manifest (first sync — everything looks “created”).

After successful propagation, updates the manifest to reflect the new synchronized state.
